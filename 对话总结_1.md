## 对话总结（供后续 AI 会话使用）

### 项目概述
- **项目名称**：FileGuard - 企业文件访问控制系统
- **目标**：构建零信任架构的文件访问控制、防泄露系统，以防止企业敏感文件被盗。以新能源汽车企业为示例场景。
- **开发语言**：Go 语言（主） + 可选 Rust（未来底层沙箱）
- **协议**：推荐 AGPL v3

### 已完成功能阶段（阶段 0 至 7）

#### 阶段0 - 基础设施
- 项目目录结构（cmd/, internal/, pkg/, api/, configs/, deployments/）
- Go module 初始化，日志库 logrus，配置 viper
- 统一错误处理、HTTP 响应封装、健康检查端点

#### 阶段1 - 存储与策略基础
- `pkg/storage`：存储接口（Get, Put, Delete, Stat, List, Move），本地文件系统实现，元数据存储为 `.meta` 文件
- `pkg/abac`：ABAC 模型（Subject, Resource, Environment, Decision），规则定义（支持正则、列表包含），内存评估器 MemoryEvaluator
- 单元测试覆盖

#### 阶段2 - 零信任网关原型
- `cmd/gateway`：HTTP 服务器，`/file/{path}` 支持 GET/PUT
- 模拟认证（从 Header 提取用户），集成 ABAC 评估
- 审计日志：`pkg/audit` 文件日志实现（JSON 行）
- 策略规则 JSON 示例（如 `rules.json`）

#### 阶段3 - 终端代理 + 水印
- `cmd/agent`：使用 fsnotify 监控目录变化，上报事件到网关
- `pkg/watermark`：图片水印（gg 库），文本文件添加注释水印
- 网关集成水印（根据策略或无条件）

#### 阶段4 - 加密与 KMS
- `pkg/crypto`：AES-256-GCM 加解密，RSA 密钥对生成
- `cmd/kms`：gRPC 服务，密钥生成、加密、解密、轮换、吊销
- 网关集成 KMS 客户端：上传时加密，下载时解密，密钥 ID 存于文件元数据
- 测试验证：文件落盘密文，下载解密正常

#### 阶段5 - DLP 与动态策略
- `pkg/dlp`：规则集 + 检测器（关键词/正则匹配），支持 action: block/alert/log
- 网关集成：上传时检测敏感内容并拦截，下载时根据敏感度强制添加水印
- 策略热加载：fsnotify 监听规则文件变化，自动重载
- 策略管理 API：CRUD 规则（`/api/policy/rules`）

#### 阶段6 - MFA (TOTP)
- `pkg/auth`：JWT 生成/验证，TOTP 密钥生成与验证
- 用户模拟存储，MFA 可启用
- 登录接口：支持密码 + 可选 TOTP，返回 JWT
- JWT 认证中间件保护 `/file/*` 路由

#### 阶段7 - 生产加固与测试
- 集成测试（模拟完整业务流程）
- 性能测试脚本（wrk）
- Docker 镜像与 Docker Compose 编排
- Kubernetes 部署示例
- golangci-lint，测试覆盖率要求 >70%
- 安全测试清单
- 文档完善

### 关键代码文件位置及作用
```
cmd/
  gateway/main.go         # 网关入口，集成所有中间件、KMS、DLP、认证
  kms/main.go            # KMS gRPC 服务
  agent/main.go          # 终端代理（监控上报）

internal/
  gateway/handler/
    file.go              # 文件 GET/PUT，加解密，水印，DLP，审计
    auth.go              # Login, SetupMFA, VerifyMFA
    policy_api.go        # 策略 CRUD
    agent.go             # 接收代理事件/心跳
  gateway/middleware/
    jwt_auth.go          # JWT 验证中间件
    logging.go
  kms/server/            # KMS 服务实现
  agent/monitor/         # 文件监控
  agent/reporter/        # 上报事件

pkg/
  storage/               # 存储接口 + 本地实现
  abac/                  # ABAC 模型，规则评估，热加载
  audit/                 # 审计日志接口 + 文件实现
  crypto/                # AES/RSA 加解密
  dlp/                   # DLP 规则集，检测器
  auth/                  # JWT, TOTP
  watermark/             # 水印叠加
  kms/                   # KMS gRPC 客户端
  config/                # 配置加载
  logger/                # 日志封装
  http/                  # HTTP 响应辅助
  errors/                # 全局错误定义

configs/
  gateway.yaml           # 网关配置：存储、策略文件、审计、KMS地址、JWT
  kms.yaml               # KMS 端口等
  agent.yaml             # 监控目录、网关地址
  rules.json             # ABAC 规则
  dlp_rules.json         # DLP 规则

api/proto/kms.proto      # gRPC 定义
```

### 主要依赖库
- logrus, viper, gorilla/mux
- fsnotify, gg (水印)
- crypto, jwt/v5, otp/totp
- grpc, protobuf

### 当前状态
- 所有阶段（0~7）均已完成测试，功能正常。
- 项目具备零信任访问控制、透明加密、水印、DLP、MFA、审计日志、热加载策略。
- 可投产，提供 Docker/K8s 部署方式。
